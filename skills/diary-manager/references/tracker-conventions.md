# Tracker 系统约定

## 文件位置

```
{{Tracker 目录}}/
├── spending.md     # 月度花销追踪
├── study.md        # 学习/考证/求职进度
├── lifestyle.md    # 养生、饮食、身体状态、运动
├── culture.md      # 书/电影/艺术展等文化消费
├── relationship.md  # 亲密关系追踪（择偶/恋爱/自我认知/关系模式）
└── todo.md         # 活跃目标和待办（替代旧的 todos/active.md）
```

## 更新时机

- **日记记录时实时同步**：云青在对话中记日记/记账时，同步更新对应 tracker 文件
- **每日提炼 cron 不更新 tracker**：只做日记提炼
- **周总结 cron（周一11:00）**：读取日记 + 全部 tracker，出周报 + 建议
- **月总结 cron（每月2号11:00）**：读取周总结 + 全部 tracker，出月报 + 深度分析

## 各文件结构

### spending.md

```markdown
# 💰 月度花销追踪

## YYYY年M月

| 日期 | 类别 | 项目 | 金额 |
|------|------|------|------|
| MM-DD | 类别 | 具体项目 | ¥XX.XX |

### 月度汇总
（按类别汇总表格）

### 按周汇总
（按周汇总表格）
```

**类别枚举**：餐饮、交通、饰品、学习、医疗、娱乐、日化、日用、数码、数字服务、居住、社交/情感、其他

**注意事项**：
- 每次追加一笔花销后，必须同步更新月度汇总的数字
- 跨天关联的开销（提前买的票等）先检查是否已在其他日期记录过
- 只记用户明确说"我花的钱"的项目
- **退货处理**：将金额改为 `~~¥XX.XX~~ 已退货`，同步扣减汇总数字
- **充值后续扣款**：若用户声明某笔充值已覆盖后续消费（如苹果账号充值），后续从该账户扣款不再重复记入开支

### 体重追踪（可选，用户启动减肥时创建）

位于 lifestyle.md 的「体重/穿戴设备」章节下，手动维护的体重曲线表。

```markdown
### 体重追踪（MM-DD 启动减肥计划）

身高：1.60m | 起始体重：69.5kg | 起始 BMI：27.1

| 日期 | 体重(kg) | BMI | 备注 |
|------|---------|-----|------|
| MM-DD | 69.5 | 27.1 | 起始 |
```

**维护规则**：
- 用户每次报体重时追加一行，自动计算 BMI（体重kg ÷ 身高m²）
- 不主动催用户上秤，只在用户主动说体重时记录
- 周总结时汇总本周体重变化趋势
- BMI 亚洲标准：≥23 超重，≥25 肥胖（比 WHO 标准更严）
```

### todo.md

```markdown
# ✅ 待办与目标追踪

## 🔄 活跃目标
（按分类：工作/求职、产品开发、学习、健康/生活、娱乐/社交、社交/情感、技术/运维）

## ✅ 已完成
（完成后移入此处，标注日期）

## ❌ 已关闭/暂停
（不再需要的项，标注关闭原因）
```

**Todo 管理规则：**
- 用户说"做完了""搞定了""不用了"时，**即时**移到已完成区或已关闭区，不能只在对话中确认
- 相关项可合并（如"供应链平台算法"+"深度讨论"→一条）
- 定期和用户 review：列出全部活跃待办，让用户逐条决定保留/完成/关闭
- 已关闭的项单独放一个区域，标注关闭原因（如"华为流程：有结果再说"）
- 日常性的行为不算待办（如"养生计划"已在 lifestyle tracker 跟踪，不需要重复列在 todo 里）

## 写入后验证（必须执行）

每次用 patch 或 write_file 修改 tracker 文件后，必须检查 markdown 表格格式：
1. **管道符检查**：每行以单个 `|` 开头，不得出现 `||` 开头的行。这是高频踩坑（culture.md 小说表头、spending.md 39行连续 `||`、study.md 表头均已出现过此 bug）。
2. **列数一致性**：表头行和数据行的管道符数量一致（4列表 = 5个 `|`）。
3. **快速验证命令**：修改后可运行 `python3 -c "lines=open('文件路径').readlines(); bad=[i+1 for i,l in enumerate(lines) if l.startswith('||')]; print(f'Bad lines: {bad}')"` 确认无格式错误。

## 设计决策

1. **Tracker 只存原始数据，不做点评**。云青的分析和建议统一在周总结和月总结中输出。
2. **Products 不单独建 tracker 文件**。已有 `products-list.md`，直接引用。
3. **旧路径 `hmWorkspace/todos/active.md` 已废弃**，统一用 `tracker/todo.md`。
